# Splitwise Clone

## Problem Statement
Managing shared expenses among friends and groups is often confusing, especially when expenses are uneven, item-based, or spread across multiple events. Existing solutions are either paid or lack transparency and flexibility.
## Features
- User authentication
- Group expense tracking
- Expense splitting
- Balance calculation

## Tech Stack
Mobile: React Native (Expo)
Backend: Node.js + Express
Database: MongoDB Atlas
Auth: JWT
Hosting: Render / Railway

## System Design Overview
User registers
→ adds friends
→ creates group
→ adds expense
→ app calculates balances
→ users view who owes whom
→ users settle
